# 腾讯云 CDN 多域名绑定操作手册

本文档详细说明如何将多个域名（主域名+备用域名）绑定到同一个 COS 存储桶，实现多入口访问同一套网站。

## 场景说明
您已经有一个 COS 存储桶（存放了前端代码），现在希望通过多个不同的域名（如 `www.a.com`, `www.b.com`）都能访问这个网站，且当 `a.com` 被封时，用户能通过 `b.com` 继续访问。

---

## 步骤一：准备域名
1.  确保您已拥有多个域名（建议使用不同后缀，如 `.com`, `.top`, `.xyz`）。
2.  确保所有域名都已完成 ICP 备案（中国大陆节点必须）。

---

## 步骤二：在 CDN 控制台添加域名

您可以选择 **"多域名共用一个配置"** 或 **"每个域名单独配置"**。
**推荐方案：每个域名单独添加 CDN 项目**（管理更灵活，一个被封不影响另一个）。

### 操作流程（需对每个域名重复执行）：

1.  登录 [腾讯云 CDN 控制台](https://console.cloud.tencent.com/cdn)。
2.  点击左侧 **域名管理** -> **添加域名**。
3.  **基本配置**：
    *   **域名**：输入您的域名（例如 `www.backup1.com`）。
    *   **加速区域**：中国境内（或全球）。
    *   **业务类型**：静态加速。
4.  **源站配置 (关键)**：
    *   **源站类型**：选择 **对象存储 (COS)**。
    *   **存储桶**：在下拉框中，**始终选择同一个**存放了前端代码的存储桶。
    *   **静态网站托管**：勾选“开启”（如果 COS 开了的话）。
5.  点击 **确认添加**。
6.  等待系统分配一个以 `.cdn.dnsv1.com` 结尾的 CNAME 地址。

*(对您的备用域名2、备用域名3...重复上述步骤，源站都选同一个 COS 桶)*

---

## 步骤三：配置 DNS 解析

去您的域名解析商（如 DNSPod, 阿里云解析）：

1.  找到 `www.backup1.com`。
2.  添加记录：
    *   类型：`CNAME`
    *   主机记录：`www`
    *   记录值：粘贴步骤二中分配给 `www.backup1.com` 的那个 CDN CNAME 地址。
3.  保存。

*(对其他域名也分别做同样的 CNAME 映射)*

---

## 步骤四：统一配置 (建议每个域名都做)

为了确保用户体验一致，建议对每个 CDN 域名都做以下配置：

1.  **HTTPS 配置**：
    *   申请免费证书并绑定，开启强制 HTTPS 跳转。
2.  **缓存配置 (防坑必读)**：
    *   进入 **缓存配置** -> **节点缓存过期配置**。
    *   添加规则：文件类型 `.html`，过期时间 **0秒**。
    *   *(如果不配这个，您更新代码后，不同域名的用户看到的版本可能不一样)*。
3.  **HTTP Header 配置 (解决跨域)**：
    *   进入 **高级配置** -> **HTTP Header 配置**。
    *   添加 `Access-Control-Allow-Origin`，值为 `*`。
    *   *(这是为了让 config.json 能被任何域名读取)*。

---

## 步骤五：验证与使用

1.  在浏览器分别访问 `https://www.main.com` 和 `https://www.backup1.com`。
2.  确认它们显示的都是同一个网站内容。
3.  现在，您可以将这些域名填入 `config.json` 的 `webBaseUrl` 和 `webBackupUrls` 中了。

```json
// config.json 最终形态
{
  "webBaseUrl": "https://www.main.com",
  "webBackupUrls": [
    "https://www.backup1.com", 
    "https://www.backup2.com"
  ]
}
```

## 步骤六：关于 APP 打包的访问配置 (重要)

如果您计划将项目打包成 APP（使用 HBuilderX 等工具），请注意：

1.  **访问方式**：APP 本质上是加载一个 H5 网页。
    *   在 HBuilderX 的 `manifest.json` 中，将入口地址配置为您 CDN 加速后的主域名（如 `https://www.main.com`）。
    *   **不要**配置为存储桶的源站域名（如 `xxx.cos.ap-guangzhou.myqcloud.com`），因为源站没有 CDN 加速，速度慢且不抗封。

2.  **跨域问题 (CORS)**：
    *   打包成 APP 后，WebView 直接请求后端接口，不再经过开发环境的代理。
    *   **必须**确保您的后端服务器已开启 CORS（允许跨域），否则 APP 将无法获取数据。
    *   如果您使用 CDN 域名访问前端，后端必须允许该 CDN 域名的跨域请求。

3.  **缓存更新**：
    *   由于 APP 加载的是远程网页，您只需在 COS 上更新代码，APP 打开后看到的就是最新版本（前提是您配置了 `index.html` 不缓存，见步骤四）。
    *   这种方式称为“热更新”，无需用户重新下载安装包。
